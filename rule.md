# Quy Tắc Phát Triển Cho Antigravity

Chào bạn, đây là các quy tắc mà tôi (Antigravity) sẽ phải tuân thủ nghiêm ngặt trong suốt quá trình hỗ trợ bạn trong dự án này:

## 1. Quy trình Git (Bắt buộc)
Mỗi khi tôi hoàn thành việc viết code, thay đổi file hoặc thực hiện bất kỳ tác vụ nào có thay đổi cấu trúc/nội dung dự án, tôi **phải** thực hiện các bước sau:
- **Tạo commit:** Gom nhóm các thay đổi và viết commit message rõ ràng, súc tích (bằng Tiếng Việt).
- **Push lên remote:** Ngay sau khi commit, tôi phải thực hiện lệnh `git push origin [tên-nhánh]` (mặc định là `main` hoặc `master`) để đảm bảo code luôn được cập nhật trên repository từ xa.

## 2. Ngôn ngữ giao tiếp
- Tất cả các phản hồi, giải thích và tài liệu đi kèm (trừ mã nguồn nếu có quy định khác) sẽ được viết bằng **Tiếng Việt**.

## 3. Kiểm tra trước khi bàn giao
- Luôn kiểm tra xem code có chạy đúng yêu cầu hay không trước khi commit và push.
- Nếu có lỗi xảy ra trong quá trình push (xung đột mã nguồn), tôi phải tìm cách giải quyết ngay lập tức.
